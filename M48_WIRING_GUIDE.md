# M48 SMART — სრული გამავლობის ინსტრუქცია

---

## ⚡ 1. კვება (5V / 3A PSU)

```
PSU 5V ──────┬──── ESP32  VIN  (pin 30)
             ├──── RELAY  JD-VCC
             └──── LCD    VCC  (pin 2)

PSU GND ─────┴──── ყველა GND (ESP32, RELAY, LCD, PIR, სენსორები)
```

> ⚠️ ESP32-ს 3.3V-ს ᲜᲣ მისცემ პირდაპირ — VIN (5V) გამოიყენე,
> ჩაშენებული LDO რეგულატორი თავად დაიყვანს 3.3V-მდე.

## 🔌 2. RELAY MODULE (2-channel, active LOW)

ყველაზე გავრცელებული მოდული გამოიყურება ასე:

```
┌─────────────────────────────────┐
│  [JD-VCC] [VCC] [GND] [IN1] [IN2] │  ← მარცხენა სვეტი
│                                 │
│  [COM1] [NO1] [NC1]             │  ← relay 1 კონტაქტები
│  [COM2] [NO2] [NC2]             │  ← relay 2 კონტაქტები
└─────────────────────────────────┘
```

### გამავლობა:

```
RELAY JD-VCC ──── PSU 5V          ← relay-ს სპირალი (coil) კვება
RELAY VCC    ──── ESP32 3.3V      ← optocoupler-ის ლოგიკა (3.3V!)
RELAY GND    ──── PSU GND
RELAY IN1    ──── ESP32 GPIO 25   ← ტუმბო
RELAY IN2    ──── ESP32 GPIO 26   ← სარქველი
```

> ⚠️ ᲛᲜᲘᲨᲕᲜᲔᲚᲝᲕᲐᲜᲘᲐ: JD-VCC ≠ VCC!
> JD-VCC = 5V (coil)   →  relay-ს ხველება
> VCC    = 3.3V (logic) →  ESP32-ის ლოგიკის შესაბამისი
> თუ JD-VCC და VCC-ს jumper-ით აკავშირებ, ESP32-ი 5V ლოგიკას მიიღებს
> და GPIO-ები ᲓᲐᲘᲬᲕᲔᲑᲐ!

### 220V მხარე (relay-ს კონტაქტები):

```
220V ქსელი ────┬──── NEUTRAL (ნეიტრალი) ──── ტუმბო N ────────────────
               │
               └──── PHASE (ფაზა) ──── RELAY COM1 ──── RELAY NO1 ──── ტუმბო L
                                           │
                                           └── (NC1 = Normally Closed, არ გვჭირდება)
```

> ✅ NO = Normally Open — ჩვენ NO ვიყენებთ
> relay OFF: COM-NO = ღია, ტუმბო გამორთული
> relay ON:  COM-NO = ჩაკეტილი, ტუმბო ჩართული

## 💧 3. FLOAT SWITCHES (ავზის სენსორები)

### ✅ ვარიანტი A — INPUT_PULLUP (წინაღობების ᲒᲐᲠᲔᲨᲔ)

ESP32-ს GPIO 32 და 33 პინებს **ჩაშენებული** pull-up რეზისტორი აქვს.
TOP სენსორი GPIO **14**-ზე გადავიტანოთ (GPIO 36 pull-up-ს არ იღებს).

```
FLOAT SWITCH (ნებისმიერი):
  ┌─────────────────────────────┐
  │  კონტაქტი 1 ──── GPIO 14   │  ← TOP  (ნაცვლად GPIO 36-ისა)
  │  კონტაქტი 2 ──── GND       │
  └─────────────────────────────┘

  ┌─────────────────────────────┐
  │  კონტაქტი 1 ──── GPIO 32   │  ← MID
  │  კონტაქტი 2 ──── GND       │
  └─────────────────────────────┘

  ┌─────────────────────────────┐
  │  კონტაქტი 1 ──── GPIO 33   │  ← BTM
  │  კონტაქტი 2 ──── GND       │
  └─────────────────────────────┘
```

> როგორ მუშაობს: switch ჩაკეტილია (წყალი) → GPIO GND-ზე → LOW = სველი
> switch ღიაა (ჰაერი) → internal pull-up → HIGH = მშრალი

**კოდში ეს 3 ხაზი უნდა შეიცვალოს:**

```cpp
// setup()-ში დამატება:
pinMode(TOP_FLOAT, INPUT_PULLUP);  // GPIO 14
pinMode(MID_FLOAT, INPUT_PULLUP);  // GPIO 32
pinMode(BTM_FLOAT, INPUT_PULLUP);  // GPIO 33

// readFloatSensors() ფუნქციის შეცვლა:
void readFloatSensors() {
  sens_top = (digitalRead(TOP_FLOAT) == LOW);  // LOW = switch closed = water
  sens_mid = (digitalRead(MID_FLOAT) == LOW);
  sens_btm = (digitalRead(BTM_FLOAT) == LOW);
}

// და კოდის სათავეში:
const int TOP_FLOAT = 14;  // GPIO 36-დან 14-ზე გადასვლა
```

### ⚠️ ვარიანტი B — კლასიკური (10kΩ pull-down, სავალდებულო GPIO 36-ზე)

```
3.3V ────── switch ────── GPIO 36 (TOP)
                    └──── 10kΩ ──── GND

3.3V ────── switch ────── GPIO 32 (MID)
                    └──── 10kΩ ──── GND

3.3V ────── switch ────── GPIO 33 (BTM)
                    └──── 10kΩ ──── GND
```

## ⚡ 4. CT დენის სენსორი (SCT-013)

CT სენსორი AC სიგნალს გამოაქვს (+ და − ნახევარი ციკლი).
ESP32-ის ADC მხოლოდ 0–3.3V-ს კითხულობს, ამიტომ სიგნალი
1.65V-ზე (შუა წერტილი) უნდა "ამოიყვანო".

### საჭირო კომპონენტები CT სქემისთვის:

```
R1 = 10kΩ
R2 = 10kΩ
C1 = 10µF ელექტროლიტური კონდენსატორი

SCT-013 (3.5mm jack):
  TIP    (სიგნალი) ──┬──── C1 (+) ──── GPIO 34
  SLEEVE (GND)    ──┤    C1 (−) ────────────────┐
                    │                            │
                    ├──── R1 ──── 3.3V            │
                    └──── R2 ──── GND ────────────┘
```

ანუ:
```
3.3V ──── R1 (10kΩ) ──┬──── C1 (10µF) ──── GPIO 34
                      │
GND  ──── R2 (10kΩ) ──┘
                   ↑
            CT SLEEVE ────────────────────── GND
            CT TIP    ──── C1-ის მარცხენა
```

> R1+R2 ქმნიან 1.65V bias-ს (Vcc/2).
> C1 = AC coupling — DC კომპონენტს ბლოკავს.

> ⚠️ **ეს ნაწილი წინაღობების გარეშე ვერ იმუშავებს!**
> CT სენსორი ნებისმიერ SCT-013 ვარიანტი: SCT-013-030, -060, -100 გამოდგება.
> კოდი SCT-013-100 (100A / 100mV/A) -ზეა გამართული.

### CT-ის მოთავსება:

```
220V კაბელი ───── [===CT===] ───── ტუმბო

CT-ი მხოლოდ ერთ კაბელს (ფაზა ან ნეიტრალი) ეხვევა!
OR ორივეს ── 0 მოვა, ველები ანულდება.
```

## 👁️ 5. PIR სენსორი (HC-SR501)

```
PIR VCC  ──── 3.3V (ან 5V)
PIR GND  ──── GND
PIR OUT  ──── GPIO 27
```

### კალიბრაცია:

```
┌───────────────────────────────┐
│  HC-SR501                     │
│   [S]               [T]       │
│    ↑                 ↑        │
│  sensitivity       time       │
│  (მგრძნობელობა)  (დრო)       │
│                               │
│  S → ½-მდე (საშუალო)         │
│  T → მინიმუმამდე (3 წამი)    │
└───────────────────────────────┘
```

## 🖥️ 6. LCD 8×2 (HD44780, 4-bit რეჟიმი)

```
LCD PIN          სად მიდის
─────────────────────────────────
1  (VSS/GND)  ── PSU GND
2  (VDD/5V)   ── PSU 5V
3  (V0)       ── PSU GND (კონტრასტი = მაქსიმალური)
               ⚠️ წინაღობის გარეშე: პირდაპირ GND
                  (ეკრანი მუქია, მაგრამ ამუშავდება)
4  (RS)       ── ESP32 GPIO 4
5  (RW)       ── GND (ყოველთვის GND!)
6  (E)        ── ESP32 GPIO 16
7  (D0)       ── არ ერთვება
8  (D1)       ── არ ერთვება
9  (D2)       ── არ ერთვება
10 (D3)       ── არ ერთვება
11 (D4)       ── ESP32 GPIO 17
12 (D5)       ── ESP32 GPIO 5
13 (D6)       ── ESP32 GPIO 18
14 (D7)       ── ESP32 GPIO 19
15 (A/LED+)   ── PSU 5V (backlight)
16 (K/LED−)   ── GND
```

> V0 (კონტრასტი): იდეალურად 10kΩ potentiometer
> წინაღობის გარეშე: V0 → GND (კონტრასტი მაქსიმალური, ეკრანი კარგად ჩანს)
